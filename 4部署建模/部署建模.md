# 4. 部署建模

【要求】使用部署图描述学校教务管理系统的物理部署结构。

## 4.1 物理架构说明
教务管理系统采用典型的三层物理架构，结合外部系统集成接口，确保系统的高可用性与安全性：

1.  **表现层 (Client Tier)**：
    -   用户通过个人电脑（PC/Laptop）或移动设备（Mobile Devices）的浏览器访问系统。
    -   采用 HTTPS 协议确保数据传输加密。
2.  **业务逻辑层 (Application Tier)**：
    -   **学校应用服务器**：部署核心教务管理系统。
    -   负责处理用户请求、执行业务逻辑，并与本地及外部数据库进行交互。
3.  **数据层 (Data Tier)**：
    -   **内部数据库服务器**：存储学生信息、成绩记录、选课流水等核心私有数据。
    -   **外部课程信息系统**：学校已有的第三方系统，通过安全 API 提供实时课程目录同步。

## 4.3 节点与通信说明

| 节点类型 | 说明 | 通信协议 |
| :--- | :--- | :--- |
| **客户端节点** | PC 或手机终端，运行标准 Web 浏览器。 | HTTPS (443) |
| **应用服务器** | 运行系统子系统及其业务逻辑。 | SQL/HTTPS |
| **本地数据库** | 用于持久化本地业务数据。 | SQL over TCP/IP |
| **外部系统** | 提供原始课程数据同步。 | REST/SOAP over HTTPS |

### 网络安全措施：
-   **防火墙隔离**：应用服务器与数据库服务器、外部系统之间均应设置防火墙规则。
-   **身份认证**：所有跨节点访问均需通过身份验证。
-   **数据加密**：敏感数据（如成绩、个人信息）在传输过程中全程加密。
