## 1.2 用例规约

### 1.2.1 [UC01] 登录

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 登录 |
| **参与者** | 学生、教师、教务处、管理员 |
| **用例说明** | 用户通过输入账号和密码进入系统。 |
| **前置条件** | 用户已拥有系统账号。 |
| **后置条件** | 用户成功进入系统主界面。 |
| **基本事件流** | 1. 用户打开系统登录页面。<br>2. 用户输入账号和密码。<br>3. 系统验证账号密码有效性。<br>4. 系统根据角色权限跳转至相应界面。 |
| **备选事件流** | 3a. 账号或密码错误：系统提示错误并允许重新输入。<br>3b. 账号被锁定：系统提示联系管理员。 |

### 1.2.2 [UC02] 维护个人信息

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 维护个人信息 |
| **参与者** | 学生、教师 |
| **用例说明** | 用户更新个人联系方式、基本信息等。 |
| **前置条件** | 用户已登录系统。 |
| **后置条件** | 个人信息更新至数据库。 |
| **基本事件流** | 1. 用户进入“个人信息”页面。<br>2. 用户修改允许编辑的字段（如联系电话、家庭住址等）。<br>3. 用户点击保存。<br>4. 系统校验格式并更新数据库。 |
| **备选事件流** | 4a. 格式校验失败：系统提示具体错误原因。 |

### 1.2.3 [UC03] 课程查询与筛选

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 课程查询与筛选 |
| **参与者** | 学生、外部课程数据库（系统参与者） |
| **用例说明** | 学生通过系统获取并筛选外部数据库中的课程信息。 |
| **前置条件** | 学生已登录，外部数据库连接正常。 |
| **后置条件** | 显示符合条件的课程列表。 |
| **基本事件流** | 1. 用户进入选课大厅。<br>2. 系统向外部数据库请求课程列表。<br>3. 用户输入筛选条件（关键词、时间、教师等）。<br>4. 系统筛选并展示课程详细信息。<br>5. 系统根据推送/拉取逻辑更新本地视图。 |
| **备选事件流** | 2a. 外部数据库连接异常：系统提示正在尝试重连或提示暂无法获取信息。 |

### 1.2.4 [UC04] 选课

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 选课 |
| **参与者** | 学生 |
| **用例说明** | 学生在规定的选课时间内进行选课操作。 |
| **前置条件** | 学生已登录，当前在选课时段内。 |
| **后置条件** | 学生选课成功或加入候补名单。 |
| **基本事件流** | 1. 学生查询感兴趣的课程。<br>2. 学生点击“选课”按钮。<br>3. 系统检查选课资格（如先修课）。<br>4. 系统检查剩余名额。<br>5. 若有名额，系统将学生加入选课名单并更新余量。 |
| **备选事件流** | 3a. 不满足先修要求：系统拦截并显示原因。<br>4a. 名额已满：系统询问是否加入候补名单，若确认则加入。 |

### 1.2.5 [UC05] 退课

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 退课 |
| **参与者** | 学生、教师、教务处 |
| **用例说明** | 学生申请放弃已选报的课程。 |
| **前置条件** | 学生已成功选中该课程。 |
| **后置条件** | 课程名单更新，候补学生递补。 |
| **基本事件流** | 1. 学生查看已选课程并选择退课。<br>2. 若在截止日期前，系统直接确认退课。<br>3. 若在截止日期后，学生填写退课原因并提交申请。 |
| **备选事件流** | 3a. 审核审批：教师及教务处审批退课申请，通过后系统执行递补逻辑。 |

### 1.2.6 [UC06] 成绩录入与修改

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 成绩录入与修改 |
| **参与者** | 教师 |
| **用例说明** | 教师维护学生的平时及期末成绩。 |
| **前置条件** | 课程已结束或正在进行中，处于成绩开放录入时段。 |
| **后置条件** | 成绩暂存或提交审核。 |
| **基本事件流** | 1. 教师进入成绩录入界面。<br>2. 输入学生平时分、期末分。<br>3. 系统自动计算总评分。<br>4. 教师保存修改或提交至管理员审核。 |
| **备选事件流** | 4a. 超过录入时段：界面锁定，无法修改。 |

### 1.2.7 [UC07] 课程信息发布与管理

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 课程信息发布与管理 |
| **参与者** | 教务处、外部课程数据库 |
| **用例说明** | 完善课程大纲、教材等信息，调整课程计划。 |
| **前置条件** | 教务人员已登录。 |
| **后置条件** | 课程信息同步至本地及外部库。 |
| **基本事件流** | 1. 教务人员选择特定课程。<br>2. 补充或修改课程描述、开课时间、授课教师等信息。<br>3. 点击同步，系统更新本地并告知外部库。 |

### 1.2.8 [UC08] 用户管理

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 用户管理 |
| **参与者** | 管理员 |
| **用例说明** | 管理员对学生、教师账号及权限进行维护。 |
| **前置条件** | 管理员已登录。 |
| **后置条件** | 用户数据更新。 |
| **基本事件流** | 1. 管理员进入用户管理模块。<br>2. 执行添加、修改、删除或批量导入操作。<br>3. 系统保存并记录日志。 |

### 1.2.9 [UC09] 成绩审核

| 项目 | 内容 |
| :--- | :--- |
| **用例名称** | 成绩审核 |
| **参与者** | 管理员 |
| **用例说明** | 审核教师提交的成绩，确保准确性。 |
| **前置条件** | 教师已提交成绩。 |
| **后置条件** | 审核通过后成绩对学生可见。 |
| **基本事件流** | 1. 管理员查看待审核成绩列表。<br>2. 检查是否有异常（如大面积不及格）。<br>3. 汇总确认或打回给教师重新核实。 |
